Demo DAG and ADF Ingestion Orchestration

Goal: separate a teaching DAG for Lessons 1-12 from the production ADLS-to-dbt ingestion flow.

Lessons 1-12 can be placed behind a Snowflake task graph as a demo, but the production pipeline should start from file arrival in ADLS and end with a tested dbt Cloud mart build.

Two Different Pipelines

Pipeline Purpose Owner
Lessons 1-12 demo DAG Show how deployed Snowpark learning assets can be sequenced. Snowflake tasks, optionally triggered by ADF.
ADLS ingestion pipeline Load new JSON files, persist raw records, trigger dbt Cloud, and archive files. ADF as the outer orchestrator; Snowflake and dbt Cloud own execution.

Demo DAG For Lessons 1-12

TASK_L01_L04_DATAFRAME_CORE
  -> TASK_L05_L07_DEPLOY_AND_DEBUG
  -> TASK_L08_L12_ADVANCED_DATAFRAME_FLOW
  -> TASK_L01_L12_DEMO_DAG_DONE

This DAG records checkpoints. It does not try to run every local Python file because those lessons are learning exercises, not a single business pipeline. The script lives in sql/LESSON29.sql.

ADF Demo Layer

For a demo, ADF can call the Snowflake root task:

execute task ORCHESTRATION.TASK_L01_L04_DATAFRAME_CORE;

Then ADF can poll Snowflake task history or read the audit table:

select *
from SNOWPARK_SAMPLE_DATA.ORCHESTRATION.LESSON_RUN_AUDIT
order by CREATED_AT desc;

Production ADLS Ingestion

ADLS car-sales/landing/
  -> ADF detects file
  -> Snowflake COPY INTO RAW.CAR_SALES
  -> dbt Cloud job builds STAGING and MART
  -> ADF moves file to archive or rejected

Use persistent raw staging for the production path. The raw table should store the JSON payload plus file and batch metadata, because CDC, replay, and audit all depend on that metadata.

Decision Rule

Question Answer
Should ADF call every Lesson 1-12 script? No. Use one root task only for the orchestration demo.
Should Snowpipe be the first production design? No. Start with ADF calling Snowflake COPY INTO for deterministic control.
Where does CDC belong? Persist raw records first; then apply row hash, source key, and incremental mart logic.
Where does the production mart build live? In dbt Cloud, not in the Lesson 16 prototype SQL.
