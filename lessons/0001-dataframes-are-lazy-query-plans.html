DataFrames Are Lazy Query Plans

Goal: read Snowpark DataFrame code and know what runs now, what only builds a plan, and where to inspect the result.

A Snowpark DataFrame is not the data itself. It is a relational query plan that Snowflake evaluates only when an action asks for results.

The Core Loop

1. Create
Point Snowpark at a table, SQL query, stage file, or small local dataset.
2. Transform
Chain methods like filter, select, group_by, and join.
3. Act
Call show, collect, count, or to_pandas to execute.

Read This Code

from snowflake.snowpark.functions import col, lit

orders = session.table("ORDERS")

big_shipped_orders = (
    orders
    .filter(col("STATUS") == lit("SHIPPED"))
    .filter(col("AMOUNT") >= 1000)
    .select("ORDER_ID", "CUSTOMER_ID", "AMOUNT")
)

big_shipped_orders.show()

session.table creates a DataFrame from a table. The two filter calls and the select call each return a new DataFrame. Snowflake executes the query when show() asks for rows.

Useful First Patterns

# Create
df = session.table("SAMPLE_PRODUCT_DATA")

# Transform
df2 = (
    df
    .filter(col("CATEGORY_ID") == 5)
    .select("ID", "NAME", col("SERIAL_NUMBER").as_("SKU"))
)

# Act
df2.show()

Use col("COLUMN_NAME") when you need comparison, arithmetic, aliasing, or function calls. Plain strings are fine for simple column selection.
