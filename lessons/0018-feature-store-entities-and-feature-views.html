Feature Store Entities and Feature Views

Goal: define reusable mortgage features with a clear entity and timestamp.

In Snowflake Feature Store, an entity is the business subject of features. A feature view is the DataFrame pipeline that computes related features for that entity.

The Mental Model

Feature store
A schema-backed feature catalog created through FeatureStore(...).
Entity
The subject and join key, such as LOAN_ENTITY with LOAN_ID.
Feature view
A registered DataFrame with features, timestamp, refresh cadence, and descriptions.

Read This Code

from snowflake.ml.feature_store import FeatureStore, FeatureView, Entity, CreationMode

fs = FeatureStore(
    session=session,
    database=DB,
    name=SCHEMA,
    default_warehouse=COMPUTE_WAREHOUSE,
    creation_mode=CreationMode.CREATE_IF_NOT_EXIST,
)

loan_entity = Entity(
    name="LOAN_ENTITY",
    join_keys=["LOAN_ID"],
    desc="Features defined on a per loan level",
)
fs.register_entity(loan_entity)

feature_df = df.select(["LOAN_ID", "TIMESTAMP", "MONTH", "DAY_OF_YEAR",
                        "DOTW", "LOAN_AMOUNT", "INCOME", "INCOME_LOAN_RATIO"])

loan_fv = FeatureView(
    name="Mortgage_Feature_View",
    entities=[loan_entity],
    feature_df=feature_df,
    timestamp_col="TIMESTAMP",
    refresh_freq="1 day",
)

fs.register_feature_view(loan_fv, version=VERSION_NUM, overwrite=True)

The lab first engineers timestamp, calendar, loan amount, income, and ratio columns, then registers them as reusable loan-level features.
