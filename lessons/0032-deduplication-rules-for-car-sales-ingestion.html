Deduplication Rules for Car Sales Ingestion

Goal: decide how duplicate files and duplicate JSON rows should behave before the mart build runs.

Raw can contain repeated load attempts. Curated staging and marts should not blindly multiply business rows when the same JSON record arrives again.

Two Duplicate Types

Duplicate typeIdentifierAction
Same file loaded againSOURCE_FILEBlock or mark duplicate batch in audit.
Same JSON row appears againSOURCE_SALE_KEYKeep one curated row for exact duplicate payloads.

Curated Staging Pattern

If raw keeps all attempts, staging can select the latest successful version per source row:

with ranked as (
  select
    *,
    row_number() over (
      partition by source_sale_key
      order by load_ts desc, load_batch_id desc
    ) as row_rank
  from {{ source('car_sales_source', 'CAR_SALES') }}
)
select *
from ranked
where row_rank = 1

Audit Gate Pattern

Before loading, ADF or Snowflake can check whether the file has already succeeded:

select count(*) as successful_file_loads
from SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_LOAD_AUDIT
where source_file = '<file_name>'
  and status = 'SUCCESS';

Important Limitation

md5(SRC::string) detects exact duplicate JSON rows. It does not detect a changed sale unless the source provides a stable business key such as sale_id.
