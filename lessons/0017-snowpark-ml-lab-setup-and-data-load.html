Snowpark ML Lab Setup and Data Load

Goal: understand the lab control plane before training any model.

The lab starts by binding the notebook session to a role, database, schema, and warehouse, then ensuring the raw mortgage table exists.

The Lab Map

Role
E2E_SNOW_MLOPS_ROLE owns the demo objects.
Database
E2E_SNOW_MLOPS_DB stores tables, feature store metadata, models, and monitors.
Schema
MLOPS_SCHEMA is the working namespace.
Warehouse
E2E_SNOW_MLOPS_WH runs DataFrame queries and ML steps.

Read This Code

from snowflake.snowpark.context import get_active_session

VERSION_NUM = "0"
DB = "E2E_SNOW_MLOPS_DB"
SCHEMA = "MLOPS_SCHEMA"
COMPUTE_WAREHOUSE = "E2E_SNOW_MLOPS_WH"
ROLE = "E2E_SNOW_MLOPS_ROLE"

session = get_active_session()
session.use_role(ROLE)
session.use_database(DB)
session.use_schema(SCHEMA)

try:

get_active_session() reuses the session from the Snowflake notebook. write_pandas is the bridge from local pandas data into a Snowflake table.
