Crossing the Result Boundary

Goal: choose when to keep data in Snowflake and when to bring a small result back to Python.

show(), collect(), and to_pandas() are actions. They execute the lazy DataFrame plan and cross from Snowflake into display output or local Python memory.

The Mental Model

show()
Preview rows for human inspection. Good for lessons and debugging.
collect()
Return a small result as Snowpark Row objects in Python.
to_pandas()
Return a small result as a local pandas DataFrame for charts or Streamlit widgets.
to_pandas_batches()
Use batches when a pandas result is too large to fit comfortably in memory at once.

Read This Code

from snowflake.snowpark.functions import col
from snowflake.snowpark.functions import sum as sum_

orders = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.ORDERS")

revenue_by_status = (
    orders
    .group_by("O_ORDERSTATUS")
    .agg(sum_("O_TOTALPRICE").as_("TOTAL_REVENUE"))
    .sort(col("TOTAL_REVENUE").desc())
)

# Human preview action.
revenue_by_status.show()

# Small Python result action.
rows = revenue_by_status.collect()

# Small notebook or Streamlit chart action.
chart_df = revenue_by_status.to_pandas()

The builder should create the DataFrame. The caller should decide which action is appropriate for the notebook, test, command-line script, or Streamlit app.

Boundary Rules

  • Keep broad filtering, joining, grouping, sorting, and ranking in Snowflake.
  • Use limit(...).show() for quick visual inspection.
  • Use collect() only for small results that Python code needs directly.
  • Use to_pandas() only after the DataFrame is already small enough for local memory.
  • Keep actions out of reusable builder functions unless the function name clearly says it runs an action.
