Documenting the Car Sales E2E Runbook

Goal: turn the completed E2E workflow into an operator-ready runbook.

A production workflow needs a runbook. The runbook should explain normal operation, recovery, validation, ownership, and known limitations.

Minimum Runbook Commands

-- Check files visible to Snowflake
list @SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_STAGE;

-- Check latest raw load
select *
from SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_LOAD_AUDIT
order by started_at desc;

-- Check current batch rows
select load_batch_id, count(*)
from SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES
group by load_batch_id
order by load_batch_id desc;

Known Limitation To Document

The current SOURCE_SALE_KEY is a hash of the JSON payload. It detects exact duplicate records. True CDC requires a stable upstream business key, such as sale_id, plus a separate record_hash.

Final E2E Statement

ADLS landing file
  -> ADF pipeline
  -> Snowflake RAW persistent staging
  -> dbt persistent staging
  -> dbt incremental MART
  -> audit evidence
  -> archive or rejected
