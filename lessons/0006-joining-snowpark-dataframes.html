Joining Snowpark DataFrames

Goal: combine rows from two Snowflake tables with a clear join condition and predictable output columns.

A join is still a lazy DataFrame transformation. Snowflake does not execute it until an action such as show(), collect(), or count().

The Mental Model

Left DataFrame
The main table you start from, such as ORDERS.
Right DataFrame
The lookup or detail table you attach, such as CUSTOMER.
Join condition
The rule that matches rows, such as order customer key equals customer key.
Join type
The row-preservation rule: inner, left, right, or full.

Read This Code

from snowflake.snowpark.functions import col

orders = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.ORDERS")
customers = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.CUSTOMER")

orders_with_customers = (
    orders
    .join(
        customers,
        orders["O_CUSTKEY"] == customers["C_CUSTKEY"],
        join_type="inner"
    )
    .select(
        orders["O_ORDERKEY"].as_("ORDER_KEY"),
        customers["C_NAME"].as_("CUSTOMER_NAME"),
        customers["C_NATIONKEY"].as_("CUSTOMER_NATION_KEY"),
        orders["O_ORDERSTATUS"].as_("ORDER_STATUS"),
        orders["O_TOTALPRICE"].as_("ORDER_TOTAL")
    )
    .sort(col("ORDER_TOTAL").desc())
)

orders_with_customers.show()

This joins each order to its customer, then projects only the columns the result should expose. The explicit select matters because joined tables often contain many columns and sometimes overlapping names.

Why Select After Join?

After a join, avoid carrying every source column forward by accident. A post-join select gives the output a clean contract for notebooks, Streamlit, tests, and downstream transformations.

.select(
    orders["O_ORDERKEY"].as_("ORDER_KEY"),
    customers["C_NAME"].as_("CUSTOMER_NAME"),
    orders["O_TOTALPRICE"].as_("ORDER_TOTAL")
)
