Ranking Rows With Window Functions

Goal: rank rows inside business groups without collapsing the rows into an aggregate summary.

A window function keeps the row-level shape of a DataFrame. It adds a calculation that looks across related rows, such as ranking orders inside each order status.

The Mental Model

Partition
The group of rows the window calculation can see, such as one O_ORDERSTATUS.
Order
The sequence inside each partition, such as highest O_TOTALPRICE first.
Window value
The derived value added to each row, such as STATUS_RANK.

Read This Code

from snowflake.snowpark import Window
from snowflake.snowpark.functions import col, row_number

orders = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.ORDERS")

status_window = (
    Window
    .partition_by("O_ORDERSTATUS")
    .order_by(col("O_TOTALPRICE").desc())
)

top_orders_by_status = (
    orders
    .with_column("STATUS_RANK", row_number().over(status_window))
    .filter(col("STATUS_RANK") <= 5)
    .select(
        "O_ORDERKEY",
        "O_ORDERSTATUS",
        "O_ORDERPRIORITY",
        "O_TOTALPRICE",
        "STATUS_RANK"
    )
    .sort("O_ORDERSTATUS", "STATUS_RANK")
)

top_orders_by_status.show()

This builds a rank inside each order-status bucket. Unlike group_by, the output still has order rows. The new column explains each row's position within its partition.

Window vs Group

Use group_by when you want fewer summary rows. Use a window when you want to keep the original row grain and add context from related rows.

# Summary shape: one row per status
orders.group_by("O_ORDERSTATUS").count()

# Row shape: each order can keep its own rank
orders.with_column("STATUS_RANK", row_number().over(status_window))
