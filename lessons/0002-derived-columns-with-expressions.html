Derived Columns With Expressions

Goal: add useful columns to a Snowpark DataFrame without leaving the lazy DataFrame API.

with_column(name, expression) returns a new DataFrame with one added or replaced column. The expression is still part of the lazy query plan.

The Pattern

from snowflake.snowpark.functions import col, lit, when

orders = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.ORDERS")

scored_orders = (
    orders
    .with_column("IS_BIG_ORDER", col("O_TOTALPRICE") >= lit(30000))
    .with_column(
        "PRIORITY_BAND",
        when(col("O_ORDERPRIORITY") == lit("1-URGENT"), lit("URGENT"))
        .otherwise(lit("NORMAL"))
    )
    .select(
        "O_ORDERKEY",
        "O_CUSTKEY",
        "O_TOTALPRICE",
        "O_ORDERPRIORITY",
        "IS_BIG_ORDER",
        "PRIORITY_BAND"
    )
)

scored_orders.show()

Read that chain left to right: start with a table, add a boolean column, add a label column, project the columns you care about, then execute with show().

Three Expression Building Blocks

col("X")
Reference a Snowflake column named X.
lit(5)
Create a literal value inside the SQL expression.
when(...).otherwise(...)
Create conditional logic like SQL CASE.

Add vs Replace

If the column name does not exist, with_column adds it. If the name already exists, with_column replaces that column in the returned DataFrame.

rounded_orders = orders.with_column(
    "O_TOTALPRICE",
    (col("O_TOTALPRICE") / lit(1000))
)

That example does not mutate the original orders DataFrame. It returns a new plan where O_TOTALPRICE has a new expression.
