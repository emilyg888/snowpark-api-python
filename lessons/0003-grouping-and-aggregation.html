Grouping and Aggregation

Goal: summarize Snowflake rows by a business category using group_by and agg.

Grouping changes the shape of a DataFrame: many input rows become fewer summary rows. Each output row represents one group.

The Mental Model

Group key
The column or columns that define one bucket, such as O_ORDERPRIORITY.
Aggregate
A summary calculation per bucket, such as count, sum, or avg.
Alias
A stable name for the output measure, such as TOTAL_REVENUE.

Read This Code

from snowflake.snowpark.functions import col
from snowflake.snowpark.functions import count, sum as sum_, avg

orders = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.ORDERS")

priority_summary = (
    orders
    .group_by("O_ORDERPRIORITY")
    .agg(
        count("*").as_("ORDER_COUNT"),
        sum_("O_TOTALPRICE").as_("TOTAL_REVENUE"),
        avg("O_TOTALPRICE").as_("AVG_ORDER_VALUE")
    )
    .sort(col("TOTAL_REVENUE").desc())
)

priority_summary.show()

This builds one row per order priority. For each priority, Snowflake counts orders, sums total price, and calculates average order value. show() remains the action boundary.

Why Alias Aggregates?

Without aliases, aggregate output names can be awkward, such as SUM(O_TOTALPRICE). Aliases make later code and tests predictable.

sum_("O_TOTALPRICE").as_("TOTAL_REVENUE")
