CDC and Incremental Marts for Car Sales

Goal: add row-level change handling to the mart tables without filtering ordinary staging views to changed rows only.

CDC belongs in mart build logic. Staging should expose clean, reusable source-shaped data; facts and dimensions decide what is new, changed, or already present.

dbt Incremental Rule

dbt incremental models build a table the first time, then process only rows selected by your incremental filter on later runs. dbt requires you to define the filter and, for merge behavior, the unique key.

Fact Model Pattern

{{
  config(
    materialized='incremental',
    unique_key='sale_key',
    incremental_strategy='merge',
    on_schema_change='sync_all_columns'
  )
}}

select
  sale_key,
  source_sale_key,
  load_batch_id,
  load_ts,
  sale_date,
  dealership,
  sales_person_key,
  customer_key,
  vehicle_key,
  sale_price,
  1 as vehicle_count
from prepared_fact_rows

{% if is_incremental() %}
where load_ts >= (
  select coalesce(max(load_ts), '1900-01-01'::timestamp_ntz)
  from {{ this }}
)
{% endif %}

Dimension Model Pattern

For this course, use Type 1 dimensions first: changed attributes replace older values for the same key.

{{
  config(
    materialized='incremental',
    unique_key='customer_key',
    incremental_strategy='merge',
    on_schema_change='sync_all_columns'
  )
}}
