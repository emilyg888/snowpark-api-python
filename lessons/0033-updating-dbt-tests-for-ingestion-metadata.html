Updating dbt Tests for Ingestion Metadata

Goal: add tests that prove every mart row can be traced back to an ADLS file and ADF run.

Once ingestion metadata becomes part of the contract, it must be tested. Otherwise the mart may be correct numerically but impossible to audit.

What To Test

LayerTests
SourceSRC, SOURCE_FILE, LOAD_BATCH_ID, LOAD_TS not null.
StagingKeys, array indexes, and metadata not null.
MartsUnique primary keys and relationships to dimensions.
AuditOne successful audit row per loaded file and batch.

Staging Schema Example

- name: stg_car_sales_base
  columns:
    - name: SOURCE_SALE_KEY
      data_tests:
        - not_null
    - name: SOURCE_FILE
      data_tests:
        - not_null
    - name: LOAD_BATCH_ID
      data_tests:
        - not_null
    - name: LOAD_TS
      data_tests:
        - not_null

Be Careful With Uniqueness

Do not enforce unique SOURCE_SALE_KEY directly on raw if raw stores all load attempts. Enforce uniqueness only in curated staging or mart models where duplicates have been resolved.

Command

dbt build --select source:car_sales_source+
