Starting a dbt Cloud Project for Car Sales

Goal: move Lesson 16 from one SQL script into a dbt Cloud project with sources, models, and tests.

dbt does not replace Snowflake. dbt stores the transformation logic, builds objects in Snowflake, and records lineage through source() and ref().

The Conversion

LESSON16.sql
  -> dbt_car_sales/models/sources.yml
  -> dbt_car_sales/models/staging/*.sql
  -> dbt_car_sales/models/marts/*.sql
  -> dbt_car_sales/models/marts/schema.yml

The raw table stays in Snowflake:

SNOWPARK_SAMPLE_DATA.PUBLIC.CAR_SALES

The dbt project declares it as a source:

{{ source('car_sales_source', 'CAR_SALES') }}

Your Turn

  1. Open dbt Cloud and create or open a project connected to this repository.
  2. Set the Snowflake connection database to SNOWPARK_SAMPLE_DATA.
  3. Use a development schema that can create STAGING and MART.
  4. Open dbt_car_sales/dbt_project.yml and confirm the staging and mart folders.
