Flattening Arrays From Variant Data

Goal: turn an array inside a semi-structured column into one output row per array element.

Direct path access is good when you need one known element, such as vehicle[0]. Use flatten when the array can contain many elements and each element deserves its own row.

The Mental Model

Input row
One sale record with a nested array, such as vehicle or extras.
Flatten
Explode the array into a lateral table function result with fields like VALUE.
Output rows
One row per array item, while still carrying selected fields from the original sale.

Read This Code

from snowflake.snowpark.functions import col, flatten
from snowflake.snowpark.types import StringType

car_sales = session.table("SNOWPARK_SAMPLE_DATA.PUBLIC.CAR_SALES")

sale_extras = (
    car_sales
    .select(
        col("SRC")["dealership"].cast(StringType()).as_("DEALERSHIP"),
        col("SRC")["vehicle"][0]["make"].cast(StringType()).as_("MAKE"),
        flatten(col("SRC")["vehicle"][0]["extras"])
    )
    .select(
        "DEALERSHIP",
        "MAKE",
        col("VALUE").cast(StringType()).as_("EXTRA")
    )
    .sort("DEALERSHIP", "MAKE", "EXTRA")
)

sale_extras.show()

The first select keeps useful source fields and expands the nested extras array. The second select names the flattened value as a clean output column.

When to Flatten

  • Use path access when the element position is part of the business rule.
  • Use flatten when every array element should become visible.
  • Keep only the original-row fields that explain the flattened value.
