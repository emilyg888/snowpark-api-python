Intro to Python UDFs

Goal: use a Python UDF for small row-level logic that SQL and Snowpark can call.

A UDF is for expression logic: one row supplies inputs and one value comes back. It is not the right tool for writing tables or running a pipeline.

The Mental Model

Input
SQL passes scalar values, such as one EXTRA string.
Handler
Python code transforms that one value.
Output
The UDF returns one scalar value that can be used in select or where.

Read This Code

create or replace function NORMALIZE_EXTRA(extra string)
returns string
language python
runtime_version = '3.12'
handler = 'normalize_extra'
as
$$
def normalize_extra(extra):
    if extra is None:
        return None
    return extra.strip().lower().replace(" ", "_")
$$;

select
  EXTRA,
  NORMALIZE_EXTRA(EXTRA) as NORMALIZED_EXTRA
from SNOWPARK_SAMPLE_DATA.PUBLIC.VEHICLE_EXTRAS;

The UDF does not know about DataFrames. Snowflake calls the Python handler for values passed by SQL expressions.

Stored Procedure vs UDF

Procedure
Use for commands, workflows, table refreshes, and multi-step Snowpark work.
UDF
Use for reusable row-level calculations inside SQL expressions.
