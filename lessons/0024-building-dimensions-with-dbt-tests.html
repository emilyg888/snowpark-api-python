Building Dimensions with dbt Tests

Goal: create unique dimension tables and prove their keys are usable.

In dbt, a model is just a select. dbt wraps that select and materializes it as a table or view based on project configuration.

Dimension Models

models/marts/dim_sales_persons.sql
models/marts/dim_customers.sql
models/marts/dim_vehicles.sql

Each dimension uses a deterministic hash key and one row per key:

qualify row_number() over (
  partition by customer_key
  order by customer_name, customer_phone, customer_address
) = 1

Tests

columns:
  - name: CUSTOMER_KEY
    data_tests:
      - not_null
      - unique

Your Turn

  1. Open dbt_car_sales/models/marts/schema.yml.
  2. Find the unique tests for the three dimension keys.
  3. Run dbt build --select dim_customers dim_vehicles dim_sales_persons.
