Saving DataFrames as Tables

Goal: materialize a finished Snowpark DataFrame as a Snowflake table with an explicit write mode.

A builder returns a lazy DataFrame. A writer call such as save_as_table is an action that creates or changes a Snowflake table.

The Mental Model

Build
Use the DataFrame API to filter, derive, flatten, aggregate, or sort.
Name
Choose the target table explicitly, such as PUBLIC.VEHICLE_PRICES.
Mode
Choose whether to append, overwrite, ignore, or error if the table exists.

Read This Code

from LESSON9 import build_vehicle_prices

TARGET_TABLE = "SNOWPARK_SAMPLE_DATA.PUBLIC.VEHICLE_PRICES"

def save_vehicle_prices(session, mode="overwrite"):
    vehicle_prices = build_vehicle_prices(session)
    vehicle_prices.write.save_as_table(TARGET_TABLE, mode=mode)

def main():
    session = create_session()
    try:
        save_vehicle_prices(session, mode="overwrite")
        session.table(TARGET_TABLE).show()
    finally:
        session.close()

The write step is deliberately separate from the builder. That keeps the transformation testable and makes the destructive boundary visible.

Write Modes

overwrite
Replace the target table with this DataFrame. Good for repeatable lesson outputs.
append
Add rows to the existing table. Use only when duplicates are understood.
errorifexists
Fail if the table already exists. Good when accidental replacement would be risky.
ignore
Do nothing if the table already exists. Good for idempotent setup in narrow cases.
