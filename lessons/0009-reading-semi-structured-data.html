Reading Semi-Structured Data

Goal: extract fields from nested Snowflake values without leaving the Snowpark DataFrame API.

Semi-structured columns can hold objects and arrays. Snowpark lets you traverse those values with column indexing, then cast the result into ordinary typed columns.

The Mental Model

Object field
Use col("SRC")["dealership"] to access a named field.
Array item
Use col("SRC")["vehicle"][0] to access an item by position.
Cast
Convert nested values into predictable output types for later code.

Read This Code

from snowflake.snowpark.functions import col
from snowflake.snowpark.types import IntegerType, StringType

car_sales = session.table("CAR_SALES")

vehicle_prices = (
    car_sales
    .select(
        col("SRC")["dealership"].cast(StringType()).as_("DEALERSHIP"),
        col("SRC")["vehicle"][0]["make"].cast(StringType()).as_("MAKE"),
        col("SRC")["vehicle"][0]["model"].cast(StringType()).as_("MODEL"),
        col("SRC")["vehicle"][0]["year"].cast(IntegerType()).as_("YEAR"),
        col("SRC")["vehicle"][0]["price"].cast(IntegerType()).as_("PRICE")
    )
    .sort(col("PRICE").desc())
)

vehicle_prices.show()

Each nested lookup returns a Snowpark column expression. The cast(...) calls turn nested values into columns that are easier to sort, filter, test, and display.

Why Cast?

Nested values may not behave like plain strings or numbers until you cast them. Cast near the extraction point so downstream code receives a clean table-shaped contract.
