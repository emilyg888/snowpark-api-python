Rerunning dbt Cloud After Source Redirection

Goal: prove the dbt Cloud job still works after the source moves from PUBLIC.CAR_SALES to RAW.CAR_SALES.

Source redirection is a production cutover. Treat it like a release: change, build, verify, and keep evidence.

Source Change

sources:
  - name: car_sales_source
    database: SNOWPARK_SAMPLE_DATA
    schema: RAW
    tables:
      - name: CAR_SALES

Local Validation

dbt build --select source:car_sales_source+

Use --full-refresh once if mart materializations change to incremental:

dbt build --full-refresh --select source:car_sales_source+

dbt Cloud Validation

  1. Push the source and model changes to GitHub.
  2. Open the dbt Cloud deployment job.
  3. Run the job manually.
  4. Confirm source, staging, mart, and tests all pass.
  5. Record the run id for ADF integration.

ADF Handoff

After dbt Cloud succeeds manually, ADF can trigger the job through a Web activity. The ADF pipeline should store the dbt run id in the load audit table.
