Ingestion Audit and Run Control

Goal: create an audit contract that connects ADLS files, Snowflake load batches, and dbt Cloud runs.

An E2E pipeline is incomplete until you can answer: what file loaded, when, how many rows, which dbt run consumed it, and did the run succeed?

Audit Table

create table if not exists SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_LOAD_AUDIT (
  LOAD_BATCH_ID string,
  SOURCE_FILE string,
  ADF_PIPELINE_RUN_ID string,
  DBT_CLOUD_RUN_ID string,
  STARTED_AT timestamp_ntz,
  FINISHED_AT timestamp_ntz,
  ROWS_LOADED number,
  STATUS string,
  ERROR_MESSAGE string
);

Status Values

StatusMeaning
STARTEDADF began processing the file.
LOADEDSnowflake COPY INTO succeeded.
DBT_SUCCESSdbt Cloud completed successfully.
FAILEDLoad, validation, or dbt failed.

Validation Query

select
  load_batch_id,
  count(*) as rows_loaded,
  count(distinct source_sale_key) as distinct_source_rows
from SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES
where load_batch_id = '<ADF_PIPELINE_RUN_ID>'
group by load_batch_id;

Run Control Rule

ADF should not archive a file until the audit table shows dbt success for the file's load batch. If Snowflake load succeeds but dbt fails, leave the file traceable and mark the audit row as failed.
