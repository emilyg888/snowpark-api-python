Intro to External Functions

Goal: understand when Snowflake should call an external HTTPS service instead of running logic inside Snowflake.

Snowflake calls these external functions. They are SQL functions whose work happens outside Snowflake through a secure proxy and remote service.

The Mental Model

Python UDF
Handler code runs inside Snowflake.
External function
Snowflake sends a request through an API integration to an external HTTPS endpoint.
Remote service
A cloud service receives rows, runs logic, and returns results.

Read This Shape

create or replace api integration VEHICLE_API_INTEGRATION
  api_provider = aws_api_gateway
  api_aws_role_arn = 'arn:aws:iam::123456789012:role/snowflake_external_function'
  enabled = true
  api_allowed_prefixes = ('https://example.execute-api.ap-southeast-2.amazonaws.com/prod/');

create or replace external function ENRICH_MAKE(make string)
returns variant
api_integration = VEHICLE_API_INTEGRATION
as 'https://example.execute-api.ap-southeast-2.amazonaws.com/prod/enrich-make';

select
  MAKE,
  ENRICH_MAKE(MAKE) as MAKE_PROFILE
from SNOWPARK_SAMPLE_DATA.PUBLIC.VEHICLE_PRICES;

This is a design shape, not a copy-paste deployment. Real external functions need cloud infrastructure, security configuration, endpoint testing, and governance review.

Use It Last

  • Use built-in SQL or Snowpark expressions when possible.
  • Use a Python UDF when logic can run safely inside Snowflake.
  • Use an external function only when the logic must call a remote system.
