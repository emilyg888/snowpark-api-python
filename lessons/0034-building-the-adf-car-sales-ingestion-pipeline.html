Building the ADF Car Sales Ingestion Pipeline

Goal: build the ADF control flow that loads ADLS files into Snowflake raw staging and then triggers dbt Cloud.

ADF owns file orchestration. Snowflake owns COPY INTO. dbt Cloud owns transformation and tests.

Pipeline Shape

Get Metadata: landing files
  -> If files exist
  -> Snowflake Script: COPY INTO RAW.CAR_SALES
  -> Snowflake Script: validate current LOAD_BATCH_ID
  -> Web Activity: trigger dbt Cloud job
  -> Until: poll dbt Cloud run status
  -> Copy/Delete or Move: archive or rejected

ADF Parameters

ParameterExample
containersnowpark-data
landing_pathcar-sales/landing/
archive_pathcar-sales/archive/
rejected_pathcar-sales/rejected/
load_batch_idADF pipeline run id.

Snowflake Script Activity

copy into SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES (
  SRC,
  SOURCE_FILE,
  SOURCE_SALE_KEY,
  LOAD_BATCH_ID
)
from (
  select
    $1,
    metadata$filename,
    md5($1::string),
    '<ADF_PIPELINE_RUN_ID>'
  from @SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_STAGE
)
file_format = SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_JSON_FORMAT
on_error = 'abort_statement';

Archive Rule

Only move files from landing/ to archive/ after both Snowflake load and dbt Cloud job succeed. Move to rejected/ only for confirmed bad input; transient failures should usually leave files in landing for retry.
