# Mission: Snowpark DataFrame API

## Why
Build practical Snowpark skills for local development in `/Users/emilygao/LocalDocuments/Snowpark`, especially notebooks and Streamlit apps that query and transform Snowflake data without dropping into raw SQL for every step.

## Success looks like
- Create Snowpark DataFrames from tables, SQL, and small local examples.
- Chain DataFrame transformations fluently: select, filter, derive columns, group, sort, and join.
- Know when a DataFrame is only a query plan and when Snowflake actually executes it.
- Debug DataFrame code by inspecting schemas, generated SQL, and small result samples.

## Constraints
- Use the local `snowpark-py312` conda environment.
- Keep lessons short and hands-on.
- Prefer official Snowflake documentation as the source of truth.

## Out of scope
- Snowpark stored procedures, UDFs, Snowpark ML, and deployment until the DataFrame API feels natural.
