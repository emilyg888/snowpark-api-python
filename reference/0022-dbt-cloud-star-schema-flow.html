dbt Cloud Star Schema Flow

A compact reference for converting the Lesson 16 Snowflake SQL into dbt Cloud models, tests, and jobs.

Flow

source('car_sales_source', 'CAR_SALES')
  -> stg_car_sales_base
  -> stg_car_sales_customers
  -> stg_car_sales_vehicles
  -> stg_car_sales_extras
  -> dim_sales_persons
  -> dim_customers
  -> dim_vehicles
  -> dim_extras
  -> fact_vehicle_sales
  -> bridge_vehicle_sale_extras

dbt Concepts

source
A declared raw table. Use it for SNOWPARK_SAMPLE_DATA.PUBLIC.CAR_SALES.
staging model
A thin extraction layer that flattens JSON arrays and normalizes names.
mart model
A business-facing dimension, fact, or bridge table.
test
A query that returns failing rows. Zero rows means pass.

Commands

dbt build --select source:car_sales_source+
dbt build --select staging
dbt build --select marts
dbt test --select marts
