Python Stored Procedures

Quick reference for moving Snowpark workflow code into a callable Snowflake object.

Use For

  • Pipeline steps and refresh commands.
  • Controlled table writes.
  • Work that should be called by SQL or scheduled later by a task.

Core Shape

create or replace procedure NAME()
returns string
language python
runtime_version = '3.12'
packages = ('snowflake-snowpark-python')
handler = 'handler_name'
as
$$
def handler_name(session):
    ...
    return "done"
$$;

call NAME();

Boundary Rule

A stored procedure is a command boundary. It is appropriate for table-changing work when the target table and write mode are explicit.
