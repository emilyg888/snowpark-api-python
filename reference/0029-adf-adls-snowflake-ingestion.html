ADF + ADLS + Snowflake Ingestion

A quick reference for the productionized car sales file ingestion design.

Architecture

ADLS Gen2
  car-sales/landing/
  car-sales/archive/
  car-sales/rejected/

ADF Pipeline
  Get Metadata
  Snowflake COPY INTO
  Validate raw batch
  Trigger dbt Cloud job
  Poll dbt Cloud job
  Move file to archive or rejected

Snowflake
  RAW.CAR_SALES
  STAGING views
  MART tables

Persistent Raw Table

create schema if not exists SNOWPARK_SAMPLE_DATA.RAW;

create or replace table SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES (
  SRC variant,
  SOURCE_FILE string,
  SOURCE_SALE_KEY string,
  LOAD_BATCH_ID string,
  LOAD_TS timestamp_ntz default current_timestamp()
);

File Format

create or replace file format SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_JSON_FORMAT
  type = json
  strip_outer_array = true;

COPY INTO Pattern

copy into SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES (
  SRC,
  SOURCE_FILE,
  SOURCE_SALE_KEY,
  LOAD_BATCH_ID
)
from (
  select
    $1,
    metadata$filename,
    md5($1::string),
    '<ADF_PIPELINE_RUN_ID>'
  from @SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_STAGE
)
file_format = SNOWPARK_SAMPLE_DATA.RAW.CAR_SALES_JSON_FORMAT
on_error = 'abort_statement';

CDC Rule

Identifier Meaning Use
SOURCE_FILE Which ADLS file produced the row. Audit, replay, archive decisions.
LOAD_BATCH_ID Which ADF run loaded the row. Batch validation and troubleshooting.
SOURCE_SALE_KEY Hash of the raw JSON row. Exact duplicate detection.
RECORD_HASH Hash of normalized business fields. True CDC when a stable business key exists.

ADF Pipeline Steps

  1. Get files from car-sales/landing/.
  2. Run Snowflake COPY INTO RAW.CAR_SALES.
  3. Validate the current LOAD_BATCH_ID.
  4. Trigger the dbt Cloud job that runs dbt build --select source:car_sales_source+.
  5. Poll the dbt Cloud run until success or failure.
  6. Move the processed file to archive/ or rejected/.

Ownership

System Owns
ADF File detection, control flow, retries, archive/reject movement, job triggering.
Snowflake External stage, persistent raw table, COPY INTO, optional task graph.
dbt Cloud STAGING and MART transformations, tests, lineage, deployment job history.
