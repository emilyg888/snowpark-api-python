Debugging Snowpark DataFrames

A quick reference for inspecting schema, generated SQL, query plans, and small samples.

Debugging Ladder

  1. Schema: confirm column names and types with df.print_schema().
  2. Plan: inspect the Snowflake query plan with df.explain().
  3. SQL: inspect generated SQL with df.queries.
  4. Sample: run a small action with df.limit(10).show().

Reusable Helper

def debug_dataframe(df, sample_size=10):
    df.print_schema()
    df.explain()

    for query in df.queries["queries"]:
        print(query)

    df.limit(sample_size).show()

When to Use Each Tool

Column error
Start with print_schema().
Unexpected SQL
Check df.queries.
Performance concern
Start with explain().
Result check
Use limit(...).show().

Testing Boundary

  • Builder functions should return DataFrames and avoid actions.
  • Debug helpers may call actions intentionally.
  • Unit tests should verify the builder is lazy.
  • Unit tests for debug helpers can use mocks to verify print_schema, explain, limit, and show.
