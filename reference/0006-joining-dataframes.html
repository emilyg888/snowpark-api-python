Joining Snowpark DataFrames

A quick reference for combining related Snowflake tables with Snowpark.

Core Pattern

joined_df = (
    left_df
    .join(
        right_df,
        left_df["LEFT_KEY"] == right_df["RIGHT_KEY"],
        join_type="inner"
    )
    .select(
        left_df["LEFT_COL"].as_("LEFT_COL"),
        right_df["RIGHT_COL"].as_("RIGHT_COL")
    )
)

Join Types

inner
Keep only rows with matches on both sides.
left
Keep all left rows and matching right rows when found.
right
Keep all right rows and matching left rows when found.
full
Keep rows from both sides, matched where possible.

Practical Rules

  • Use bracket references such as orders["O_CUSTKEY"] when both DataFrames are in scope.
  • Alias output columns with .as_("NAME") so tests and downstream code are stable.
  • Select after joining so the output contract is explicit.
  • Keep show(), collect(), and count() outside builder functions.
  • Start with inner joins unless you intentionally need unmatched rows.

TPCH Example

orders = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.ORDERS")
customers = session.table("SNOWFLAKE_SAMPLE_DATA.TPCH_SF1.CUSTOMER")

orders_with_customers = (
    orders
    .join(customers, orders["O_CUSTKEY"] == customers["C_CUSTKEY"])
    .select(
        orders["O_ORDERKEY"].as_("ORDER_KEY"),
        customers["C_NAME"].as_("CUSTOMER_NAME"),
        orders["O_TOTALPRICE"].as_("ORDER_TOTAL")
    )
)

Common Mistakes

  • Joining on columns that look similar but use different business meanings.
  • Forgetting to choose output columns after the join.
  • Calling show() inside a builder, which makes testing harder.
  • Using a left join when an inner join is enough.
