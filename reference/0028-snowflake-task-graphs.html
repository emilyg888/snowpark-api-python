Snowflake Task Graphs

A compact reference for scheduling Snowflake-native work from Lessons 13-16.

Core Pattern

create task ROOT_TASK
  warehouse = COMPUTE_WH
  schedule = 'USING CRON 0 7 * * * Australia/Sydney'
as
  call SOME_PROCEDURE();

create task CHILD_TASK
  warehouse = COMPUTE_WH
  after ROOT_TASK
as
  call NEXT_PROCEDURE();

Rules

Rule Use It For
Only the root task needs a schedule. Child tasks inherit timing through after dependencies.
Tasks run SQL or call stored procedures. Promote local scripts into Snowflake objects before scheduling them.
Resume children before the root. A suspended child will not run even if its predecessor succeeds.
Keep external calls optional until proven. External functions add network, IAM, endpoint, and monitoring concerns.

Lessons 13-16 Mapping

Lesson 13 procedure -> call PUBLIC.REFRESH_VEHICLE_PRICES()
Lesson 14 UDF       -> use PUBLIC.NORMALIZE_EXTRA(EXTRA) in a task query
Lesson 15 external  -> call only after API integration and endpoint are deployed
Lesson 16 SQL mart  -> wrap as MART.REFRESH_CAR_SALES_STAR_SCHEMA()

Operational Checks

  • Confirm the task owner role can use the warehouse and referenced objects.
  • Confirm the root schedule timezone and cadence match the data refresh need.
  • Query task history after the first manual or scheduled run.
  • Suspend the root before editing the graph.
