IFRS9 PROC SQL Staging To Snowflake

Goal: translate the first SAS 9 PROC SQL pattern from the synthetic IFRS9 dataset into Snowflake SQL and Snowpark.

Dataset: /Users/emilygao/LocalDocuments/Projects/bb_datasets/ifrs9. Start with the calculation shape, not the platform syntax. A SAS create table ... as select block usually becomes a Snowflake CTAS statement before it becomes a Snowpark DataFrame.

The SAS Pattern

%let as_of_dt = '31MAY2026'd;

proc sql;
  create table ifrs9.stage_result as
  select
      a.account_id,
      a.cust_id,
      a.portfolio_cd,
      a.prod_cd,
      a.drawn_amt,
      a.undrawn_amt,
      a.dpd,
      a.curr_risk_grade,
      case
        when a.default_flg = 'Y' or a.dpd >= 90 then '3'
        when a.dpd >= 30 or a.forbearance_flg = 'Y' then '2'
        else '1'
      end as ifrs9_stage length=1,
      a.as_of_dt
  from crdmart.loan_account a
  where a.as_of_dt = &as_of_dt;
quit;

Snowflake SQL First

create or replace table IFRS9.STAGE_RESULT as
select
  ACCOUNT_ID,
  CUST_ID,
  PORTFOLIO_CD,
  PROD_CD,
  DRAWN_AMT,
  UNDRAWN_AMT,
  DPD,
  CURR_RISK_GRADE,
  case
    when DEFAULT_FLG = 'Y' or DPD >= 90 then '3'
    when DPD >= 30 or FORBEARANCE_FLG = 'Y' then '2'
    else '1'
  end as IFRS9_STAGE,
  dateadd(day, AS_OF_DT, date '1960-01-01') as AS_OF_DT
from IFRS9.LOAN_ACCOUNT
where dateadd(day, AS_OF_DT, date '1960-01-01') = date '2026-05-31';

The important migration decision is boring and valuable: keep relational SQL as relational SQL first. This lets you reconcile row counts, exposure totals, stage distributions, and SAS numeric date conversion before introducing a new abstraction.

Snowpark Sample

from snowflake.snowpark.functions import col, dateadd, lit, when

stage_base = (
    session.table("IFRS9.LOAN_ACCOUNT")
    .with_column("AS_OF_DATE", dateadd("day", col("AS_OF_DT"), lit("1960-01-01")))
    .filter(col("AS_OF_DATE") == lit("2026-05-31"))
    .select(
        col("ACCOUNT_ID"),
        col("CUST_ID"),
        col("PORTFOLIO_CD"),
        col("PROD_CD"),
        col("DRAWN_AMT"),
        col("UNDRAWN_AMT"),
        col("DPD"),
        col("CURR_RISK_GRADE"),
        when((col("DEFAULT_FLG") == lit("Y")) | (col("DPD") >= 90), lit("3"))
        .when((col("DPD") >= 30) | (col("FORBEARANCE_FLG") == lit("Y")), lit("2"))
        .otherwise(lit("1"))
        .alias("IFRS9_STAGE"),
        col("AS_OF_DATE").alias("AS_OF_DT"),
    )
)

stage_base.write.mode("overwrite").save_as_table("IFRS9.STAGE_RESULT_SP")

Translation Moves

SAS Snowflake Reason
ifrs9.stage_result IFRS9.STAGE_RESULT The dataset lineage maps SAS IFRS9 output tables to Snowflake target objects.
'31MAY2026'd dateadd(day, AS_OF_DT, date '1960-01-01') The CSV stores SAS numeric dates, so convert from the SAS epoch.
case when case when No need to change a clear SQL expression.
default_flg and forbearance_flg DEFAULT_FLG and FORBEARANCE_FLG Keep the rule inputs visible so stage allocation can be reviewed.
