IFRS9 Stage Allocation Logic

Goal: move stage allocation rules from SAS SQL into reviewable Snowflake SQL.

Keep rule order explicit. Stage 3 rules usually need to win before Stage 2 rules.

Core Example

case
  when DEFAULT_FLAG = 'Y' or DAYS_PAST_DUE >= 90 then 3
  when SICR_FLAG = 'Y' or DAYS_PAST_DUE >= 30 then 2
  else 1
end as IFRS9_STAGE

Hands-On Exercise

  1. Write the stage rule as a truth table.
  2. Translate it into case when.
  3. Aggregate EAD by stage and compare to SAS.
