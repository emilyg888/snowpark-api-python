IFRS9 Exposure Joins

Goal: translate joins across loan account, customer, collateral, and parameter tables into Snowflake SQL with row-count controls.

A join migration is not done when the query runs. It is done when key uniqueness and row inflation are explained.

Core Example

create or replace temp table IFRS9_EXPOSURE_JOINED as
select
  a.ACCOUNT_ID,
  a.CUST_ID,
  c.CUST_SEGMENT,
  a.PORTFOLIO_CD,
  p.PD_12M,
  p.LGD_BASE
from IFRS9.LOAN_ACCOUNT a
left join IFRS9.CUSTOMER c on a.CUST_ID = c.CUST_ID
left join IFRS9.PD_LGD_PARAMETER p
  on a.PORTFOLIO_CD = p.PORTFOLIO_CD
 and a.CURR_RISK_GRADE = p.RISK_GRADE;
