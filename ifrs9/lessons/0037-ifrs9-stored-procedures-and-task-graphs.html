IFRS9 Stored Procedures And Task Graphs

Goal: package reviewed migration steps into callable Snowflake objects and task dependencies.

A task should call a stable Snowflake object or run one clear SQL statement. Do not schedule a local SAS script.

Core Example

create or replace task IFRS9_RUN_STAGE_BASE
  warehouse = COMPUTE_WH
  schedule = 'USING CRON 0 6 * * * Australia/Sydney'
as
  call IFRS9_REFRESH_STAGE_BASE();

Hands-On Exercise

  1. Wrap the staging query in a stored procedure.
  2. Create a root task for the cutoff run.
  3. Add a child task for reconciliations.
